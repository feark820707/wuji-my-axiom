# PATCH-001：貞的接地修正

> 版本：v0.1 → v0.2
> 日期：2026-04-15
> 嚴重度：重大 — 不修此項，所有爻則形同裝飾

---

## 發現了什麼問題

爻則寫著「沒驗證的不算數」，但 AI 從未自動啟動這條規則。

實際案例：AI 從一段未驗證的描述（「預測餵食量」）出發，推導出完整的控制流程設計 — 狀態機、資料表、LED 映射，全部邏輯自洽，但**事實是錯的**（實際輸出是馬達轉速 0-1023，不是克數，硬體也沒有秤）。

Owner 糾正四次，AI 才找到根因。

## 為什麼會這樣

框架假設 AI 具備**元認知** — 能自行判斷「這是已驗證的事實」還是「這是我自己推導出來的合理文字」。

LLM 沒有這個能力。對它來說，自己生成的文字和外部事實看起來一模一樣。

結果：
- 推導鏈越長，每一步都「聽起來合理」
- 合理感累積 = 幻覺長鞭
- 最終產出邏輯自洽但離事實越來越遠
- 爻則存在但不啟動，因為 AI 不知道自己在用未驗證的前提

## 怎麼修

不是再加一條爻則（那又是被動規則）。是改**生生循環本身**。

### 改動 1：貞 = 接地，不是自我比對

舊版：
```
貞而無據，標記不確定，交人。
```

新版：
```
貞而無據，標記不確定，交人。
貞必接地 — 比對外部事實（原始碼、文件、數據），不是比對自己的推導。
```

### 改動 2：新增爻則 — 來源標記

在爻則區塊最後加一行：

```
事實主張必須附來源，無來源 = 未驗證 = 不能作為決策輸入。
```

### 改動 3：新增爻則 — 動手前先挖根因

```
動手前先挖根因：移除此因問題是否消失（But-For）、只修此因是否不復發（復發測試）、此因能否解釋所有症狀（解釋測試）。三項未過不算找到根因。
```

## 怎麼更新

1. 重新複製 `templates/CLAUDE.md` 覆蓋你的舊版
2. 把你的「我的卦」（兩儀、四象、應用層太極）貼回去
3. 完成

「我的卦」是你個人校準的結果，框架更新不影響它。其餘部分（生生、爻則）已在新模板中修正。
