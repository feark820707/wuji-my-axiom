# 系統架構

> 道不動、德常新、器可換。

## 三層結構

```
┌─────────────────────────────────────────┐
│  道（CLAUDE.md）                         │
│  太極 + 生生 + 爻則 + 我的卦 + 變通      │
│  ─── 不動層：定義系統的根本行為 ───       │
├─────────────────────────────────────────┤
│  德（Memory）                            │
│  回饋記憶 + 使用者畫像 + 專案狀態         │
│  ─── 常新層：使用中持續累積 ───           │
├─────────────────────────────────────────┤
│  器（Skills / Tools）                    │
│  具體工具、腳本、專案套件                  │
│  ─── 可換層：隨時替換不影響系統 ───       │
└─────────────────────────────────────────┘
```

### 道層（不動）

檔案：`CLAUDE.md`

- **太極**：一條公理，定義使用者的核心價值
- **生生**：感→化→貞→新循環，定義行為節奏
- **爻則**：邊界條件，定義不可違反的限制
- **我的卦**：冷啟動校準產出，定義使用者的個人特質
- **變通**：校準規則，定義系統如何從回饋學習

道層極少變動。只有紀元轉換（見 [EVOLUTION.md](EVOLUTION.md)）才會修改。

### 德層（常新）

檔案：`MEMORY.md` + `memory/*.md`

- 使用者回饋（糾正與確認）
- 使用者偏好與習慣
- 專案狀態與脈絡
- 參考資料索引

德層每次對話都可能更新。是系統的活記憶。

### 器層（可換）

檔案：具體的工具、腳本、專案檔案

- 不屬於框架本身，屬於使用者的工作環境
- 替換任何工具不影響道與德
- 道和德決定如何使用器，器不反過來約束道德

## 生生循環中的三層互動

```
感（收到刺激）
 │
 ├─ 讀取道層：太極 + 爻則 → 推導行動方向
 ├─ 讀取德層：記憶 → 補充個人脈絡
 │
化（執行行動）
 │
 ├─ 調用器層：工具 + 腳本 → 產出結果
 │
貞（校驗結果）
 │
 ├─ 接地驗證：比對外部事實（原始碼、文件、數據），不是比對自己的推導
 ├─ 比對道層：結果是否符合太極與爻則
 ├─ 無來源的事實主張標記 [未驗證]，不能作為決策輸入
 ├─ 標記不確定的交人
 │
新（更新系統）
 │
 ├─ 寫入德層：記憶更新
 ├─ 極少數情況更新道層：爻則微調
 └─ 器層按需調整
```

## 普遍 / 個人 / 湧現

| 類型 | 來源 | 範例 | 生命週期 |
|------|------|------|----------|
| 普遍（預設） | 框架內建 | 生生循環、變通規則 | 永久 |
| 個人（校準） | 冷啟動校準 | 我的卦、應用層太極 | 長期穩定 |
| 湧現（演化） | 使用中累積 | 回饋記憶、行為模式 | 持續變化 |

冷啟動校準把普遍框架轉化為個人系統。使用中的變通讓個人系統持續演化。

## 變通機制

回饋的流向：

```
使用者糾正/確認
       │
       ▼
  德層（記憶）── 累積 ──→ 模式浮現
       │                      │
       │                      ▼
       │               爻則候選
       │                      │
       │              穩定驗證通過
       │                      │
       ▼                      ▼
  德層更新              道層更新（罕見）
```

大部分回饋停留在德層。只有反覆出現、穩定不變的模式才會升級為爻則，寫入道層。這確保道層的穩定性。

## 設計原則

1. **上層約束下層**：道約束德，德約束器。不反轉
2. **下層不污染上層**：器的變動不影響德，德的變動極少影響道
3. **最小載入**：每次啟動只載入需要的部分，不吃進全部記憶
4. **靜默正常**：不需要報告的就不報告，只報異常
