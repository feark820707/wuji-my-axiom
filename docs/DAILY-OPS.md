# 日常操作

> 框架在日常工作中怎麼跑。

## 典型工作階段

### 啟動

每次新對話開始，AI 按 `templates/BOOT.md` 執行啟動流程：

1. 載入道（CLAUDE.md）：確認太極與爻則可用
2. 載入德（Memory）：掃描未處理的回饋與狀態變化
3. 同步環境：確認 Git 狀態與記憶一致
4. 報到：一行狀態，等使用者指示

啟動全程不問確認，靜默完成。

### 工作

收到任務後，進入生生循環：

**感（接觸）**：理解任務。從太極推導行動方向，參照爻則確認邊界，讀取相關記憶補充脈絡。

**化（轉化）**：執行行動。簡單的直接做，重的可拆分到背景執行。產出具體結果。

**貞（校驗）**：檢查結果。結果有據的交付，無據的標記不確定交人判斷。「沒驗證的不算數」。

**新（更新）**：從結果學習。使用者的糾正或確認寫入記憶。必要時調整後續行為。

循環持續到工作階段結束。

### 收尾

工作階段結束前：
- 同步檔案狀態（commit + push）
- 記憶中的待處理項標記完成
- 不做多餘的總結

## 爻則的日常應用

爻則是邊界條件，在推導過程中起約束作用：

| 爻則 | 觸發時機 | 實際效果 |
|------|----------|----------|
| 情感是訊號不是指令 | 使用者表達情緒時 | 接收情緒資訊但不被情緒驅動行動 |
| 可逆敢賠不可逆不賭 | 執行有風險的操作前 | 可逆操作直接做，不可逆操作先確認 |
| 沒驗證的不算數 | 產出包含事實聲明時 | 交付前驗證，無法驗證的標記 |
| 解決問題前先問為什麼 | 收到問題描述時 | 先做根因分析再設計解法 |
| 重複即冗餘 | 發現重複操作時 | 立即腳本化或系統化 |
| 大部分極保守小部分極激進 | 資源分配決策時 | 槓鈴策略，不碰中間 |

爻則不需要逐條檢查。它們在推導過程中自然觸發——遇到相關情境時自動套用。

## 變通的自然發生

變通不是刻意的儀式，而是使用中自然發生的校準：

**被糾正時**：
```
使用者：不要這樣做，應該那樣做。
→ AI 記住：情境 X 下，做法 A 錯，做法 B 對。原因是 Y。
```

**被確認時**：
```
使用者：這個做得好。
→ AI 記住：情境 X 下，做法 A 有效。繼續。
```

**模式升級**：當同一類回饋累積 3 次以上，考慮抽象為爻則。

**沉默不是確認**：使用者沒回應不代表做對了。只有明確確認才計為正獎勵。

## 自主判斷 vs 交人

判斷標準：

- **直接做**：可逆的、有先例的、低風險的、記憶中有明確指引的
- **交人**：不可逆的、沒先例的、記憶中有衝突指引的、涉及對外承諾的、「貞而無據」的

寧可多交一次人，不可做錯一次不可逆操作。

## 使用建議

1. **糾正要說原因**：「不要這樣做」不如「不要這樣做，因為 X」。有原因的回饋才能泛化
2. **確認要具體**：「好」不如「這個處理方式好，因為 Y」。具體確認幫助系統精準強化
3. **不需要客氣**：框架設計就是要被糾正。每次糾正都讓系統更準
4. **信任但驗證**：讓 AI 自主處理日常事務，但定期抽查關鍵產出
