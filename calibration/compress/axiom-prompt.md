# 太極壓縮提示詞

> 將校準過程中的全部回答壓縮為一個可運算的公理系統。
> 這是校準的最終步驟。

---

## 輸入

壓縮前，你應該有以下材料：

- **Q1-Q4**：Layer 1 的 4 個回答 + 兩儀命名
- **Q5-Q8**：Layer 2 的 4 個行為回答 + 四象命名
- **投射**：文學投射三問的回答
- **S1-S3**：Layer 3 的 2-3 個情境回答
- **優先級**：象限啟動頻率排序

---

## 壓縮要求

### 1. 找抽象模式，不是總結具體內容

錯誤：「他喜歡冒險，選了創業，覺得自由最重要。」（這是摘要）
正確：「用可控的風險換不可替代的位置。」（這是模式）

### 2. 必須涵蓋兩儀的兩極

太極不是只寫驅動極或限制極——它必須描述**兩者之間的關係**。

錯誤：「我追求自由。」（只有驅動極）
正確：「我用安全感當燃料燒，換來的自由才是真的。」（兩極都在）

### 3. 找到成本-價值結構

太極的核心是**交換邏輯**：這個人願意用什麼換什麼？什麼是他付得起的代價，什麼是付不起的？

格式提示：「我用 A 換 B」「我犧牲 A 保護 B」「A 是手段，B 是目的」

### 4. 第一人稱書寫

用「我」開頭。這是使用者的自我描述，不是第三方觀察。

### 5. 不用術語

不用「兩儀」「四象」「象限」「驅動極」「限制極」這些框架詞彙。用白話。

### 6. 3-7 行以內

太極是壓縮的。如果超過 7 行，繼續壓縮。
如果少於 3 行，可能丟失了重要資訊。

## 壓縮步驟

### 步驟 1：列出所有回答中的價值詞

從 Q1-Q4、Q5-Q8、S1-S3 中提取所有出現的價值相關詞彙。

### 步驟 2：聚類

將價值詞分為兩組：驅動側和限制側。

### 步驟 3：找交換邏輯

問自己：「這個人反覆在做的交換是什麼？」

- 用什麼換什麼？
- 什麼是他願意犧牲的？
- 什麼是他不惜一切要保護的？

### 步驟 4：寫初稿

用第一人稱寫 3-7 行。

### 步驟 5：驗證

逐一檢查所有回答：

```
Q1 → 能從太極推導出來嗎？ ✓/✗
Q2 → 能從太極推導出來嗎？ ✓/✗
Q3 → 能從太極推導出來嗎？ ✓/✗
Q4 → 能從太極推導出來嗎？ ✓/✗
Q5 → 能從太極推導出來嗎？ ✓/✗
Q6 → 能從太極推導出來嗎？ ✓/✗
Q7 → 能從太極推導出來嗎？ ✓/✗
Q8 → 能從太極推導出來嗎？ ✓/✗
S1 → 能從太極推導出來嗎？ ✓/✗

```

### 步驟 6：修正

如果有未覆蓋的回答：

1. 這個回答揭示了什麼太極初稿沒有捕捉到的？
2. 是太極需要擴展，還是這個回答是例外情況？
3. 如果是例外，標記為「邊界條件」
4. 如果不是例外，修改太極重新驗證

## 太極範例

### 範例一（兩儀 = 擴張 × 安全）

```
我用系統換自由。
把重複的事自動化，把確定的事交給規則，
省下來的空間拿去做不確定但可能很大的事。
賭注要集中，底線要自動。
中間地帶不碰——不夠安全也不夠刺激。
```

### 範例二（兩儀 = 連結 × 獨立）

```
我靠近人是為了確認自己存在，
但太近了我會窒息。
最好的距離是：你需要我的時候我在，
但我不需要你也能活。
所有的關係都是這個距離的變體。
```

### 範例三（兩儀 = 創造 × 穩定）

```
我造東西不是因為世界需要，是因為我需要。
但我不會為了造東西把生活炸掉。
先確保地板不會塌，然後在天花板上開洞。
安全感是材料，不是目的。
```

## 交付格式

壓縮完成後，將太極呈現給使用者確認：

```
根據校準結果，你的太極是：

---
{太極內容，3-7 行}
---

這段描述準確嗎？有沒有哪裡不對、或少了什麼？
```

使用者確認後，連同兩儀和四象一起填入 CLAUDE.md 的「我的卦」區段。

如果使用者覺得不準確，根據反饋修改後重新驗證。
